# SRPG Proto 2.0 — エンジン設計

## プロジェクト概要
グノーシア×FE×スレスパをコンセプトとしたローグライクSRPG。ブラウザ自作（Vite + React）。
1280×720固定解像度、48pxタイル。個人開発プロジェクト。

- 会話はタメ口（敬語禁止）

## ゲームの主要ルール

### 戦闘
- ダメージ = Y + X（Y=PP+技威力, X=攻STR-防DEF）
- 技type体系: flatAdd(ATK+mod-DEF), mult(ATK×mult-DEF), fixed(DEF無視), intRef(INT×mult-敵INT)
- 追撃 = INT差%の事故枠（INT差1で1%）。自ターンのみ、プレーン射程内のみ
- 処理順: 攻撃→反撃→追撃→デバフ適用
- 反撃はプレーン攻撃のみ（技不可）
- デバフ命中 = 基礎命中% + (自INT-敵INT)×5%
- ブレイク/シール/バインド = ダメージなし

### 経験値・成長
- 経験値 = プール方式（敵1体上限20、ヒット5、トドメで残り総取り）
- 必要EXP: 固定20/Lv
- 成長率: クラス依存の固定値。端数蓄積・切り捨て

### クラスチェンジ（重要: マップ間イベント）
- **CCはレベルアップ時ではなく、マップ間のインターバルで行う**
- CCは成長枠を消費しない（ステ補正+PP加算のボーナスのみ）
- M2クリア後のインターバル → 中級CC選択（2択）
- M4クリア後のインターバル → 上級CC選択（2択）
- CC時に特殊ドラフト抽選（選んだタグ100%/U率UP）

### クラスレベル制
- 初級3回成長 + 中級5回成長 + 上級3回成長 = 11回
- 通しレベルで管理: 初級Lv1-3, 中級Lv4-8, 上級Lv9-11
- レベルキャップ13（11回成長 + CC2回でLvは上がるが成長なし）

### パーティ・ループ構造
- 13人プール → 初期6人ランダム(Lv1) → M1後に残7から3提示2選択 → 8人固定
- ループ = 5マップ + ラスボス（5+1構成）
- ユニットロスト: マップ間で30%HPで復活 + 経験値ペナルティ
- 全員出撃8人制（ベンチなし）

### レベル進行
| マップ | 敵数 | 人数 | 進行 |
|--------|------|------|------|
| M1 | 6 | 6 | 初級+1 |
| M2 | 16 | 8 | 初級+2（初級完了）|
| インターバル: 中級CC | | | |
| M3 | 24 | 8 | 中級+3 |
| M4 | 24 | 8 | 中級+2（中級完了）|
| インターバル: 上級CC | | | |
| M5 | 24 | 8 | 上級+3（上級完了）|
| M6(ラスボス) | 自由 | 8 | キャップ |

### ドラフト
- レベルアップ時に3択パーツ提示
- パーツ種別: tech / skill の2種（旧itemはskillに統合）
- レアリティ: C65% / U28% / R7%
- タグ重み: 得意A 38% / 得意B 38% / 苦手 12% / 汎用(光闇) 12%
- 同一パーツ複数取得可能（初期技含む。1回の3択内での重複のみ除外）

### アイテムボックス
- マップ上にアイテムポイント配置。ユニットが乗ると拾える
- 敵がアイテムボックス地形に待機すると消滅
- アイテム: 傷薬(HP15)/キノコ(MOV+5,1T,行動消費なし)/羽(飛行3T,行動消費なし)/バフ薬STR+2/DEF+2/INT+2(マップ中)/ゴールド

### デバフ体系
| デバフ | 効果 | 命中 |
|--------|------|------|
| 目まい(旧ブレイク) | 反撃封じ(NT) | INT判定 |
| 忘却(旧シール) | 技封じ(NT) | INT判定 |
| 捕縛(旧バインド) | 移動封じ(NT) | INT判定 |
| ATK低下 | ATK-3(1T) | 必中 |
| DEF低下 | DEF-3(1T) | 必中 |
| タゲ固定 | 1T | 必中 |
| 火傷 | 固定6ダメ/ターン終了時 | 技依存 |

ボスランク: 捕縛→MOV半減、忘却→チャージ技不干渉

### 兵種特性
| クラス | 特性 |
|--------|------|
| スレイヤー | 攻撃後に再移動（canto） |
| ヴァンガード | 被ダメ返しターンATK+3（revenge） |
| フォートレス | HP0攻撃を1回耐える（miracle） |
| プロテクター | 隣接味方への攻撃を肩代わり（substitute） |
| オラクル | 回復+50%（healBoost） |
| ソーサラー | 射程2↑技ATK+2/妨害命中+15（rangeBoost） |

## UI操作フロー（状態遷移）

### プレイヤーフェーズの操作フロー
```
[select] ← ユニット選択待ち。敵クリックでマーキング。右クリックでコンテキストメニュー
  │ 味方ユニット左クリック
  ▼
[select + moveCells] ← 移動先選択。右クリックでキャンセル
  │ 移動先クリック
  ▼
[action] ← アクションメニュー。「戻る」でorigPosに戻れる
  ├─ 通常攻撃/行動消費技 → ターゲット選択 → 実行 → acted=true
  ├─ 待機 → acted=true
  ├─ 戻る → origPosに戻す → [select]
  └─ noAction技 → 技消費 → 下へ
        ▼
      [select + moveCells(残MOV)] ← 再移動先選択。「戻る」なし
        │ 移動先クリック or 範囲外クリック
        ▼
      [action] ← 「戻る」で再移動前(noActionPos)に戻れる
        ├─ 通常攻撃/行動消費技/待機 → acted=true
        ├─ 戻る → noActionPosに戻す → 再移動画面
        └─ noAction技 → さらに繰り返し可能
```

### noAction技のルール
- 行動消費しない（=ターンを終了しない）だけ。移動力は戻らない
- 各noAction技は1ターンに1回ずつしか使えない（oncePer）
- 使用後、残り移動力分だけ再移動できる
- noAction技使用前の位置には戻れない。使用後の再移動のみ戻れる

### 敵フェーズ
- 1体ずつ逐次で計画→実行。先に動いた敵の位置を次の敵が参照する
- ENEMY PHASE バナー表示 → 各敵行動 → ターン終了処理 → PLAYER PHASE バナー

### 右クリック
- 空セル: コンテキストメニュー（ターン終了/マーキング解除）
- ユニット: ステータス表示
- アクション中/ターゲット選択中: キャンセル（右クリック=戻る）

### 敵マーキング
- 敵を左クリックでトグル。複数敵同時マーキング可能
- マーキング敵のユニット上に◎表示
- 敵の移動範囲+攻撃範囲を合算して赤セルで表示
- unitsが変化したら自動再計算（敵死亡で消滅）

### ホバーツールチップ
- ユニットにマウスホバーでミニステータス表示（名前/Lv/HP/デバフ）
- **非表示条件**: アクションメニュー表示中、コンテキストメニュー表示中、ステータス画面表示中
- **位置**: ユニットの右横。画面右端に近ければ左側に回り込む。画面下端に近ければ上にずらす
