# お祭りテスト — デバッグ編成投入口

> 企画の生死判定用の**使い捨て検証ツール**。ボツになったら `src/debug/` ごと、
> App.jsx の `FESTIVAL_MODE` 周りと `src/maps/festival.json` を削除すれば消える。

ドラフト・CC・レベルアップを一切経ずに、**任意パーツ装備済み・育成完了の8体**を
お祭り面に直置きして即戦闘に入る。

## 起動方法

どちらでもOK（UIは作っていない）：

- URL に付ける: `http://localhost:5173/?festival`（`?fest` でも可）
- コンソール: `__debug.festival()`（通常起動後でも途中から差し込める）

起動すると配置フェーズもスキップして、8体がデプロイゾーン先頭から自動配置され、
そのまま自ターンに入る。

## 構成ファイル

| ファイル | 役割 |
|----------|------|
| `src/debug/festivalRoster.json` | デバッグ編成8体（手書き）。ここを書き換えてビルドを差し替える |
| `src/debug/festival.js` | ロスター → 戦闘可能ユニット化する口（実装本体） |
| `src/maps/festival.json` | お祭り面（**仮**。マップエディタ出力で差し替える前提） |
| `src/App.jsx` | `FESTIVAL_MODE` 判定 + `startFestivalStage()` + `__debug.festival()` |

## ロスター形式（①最終ステ直書き）

```jsonc
{
  "charId": "p01",          // units.json の id。基礎データ（名前/タグ/色/個人特性）を引く
  "label": "役割メモ",       // ローダーは無視。可読用
  "class": "スレイヤー",      // 最終クラス。兵種特性(canto等)・射程・枠はこれで解決
  "level": 13,              // 表示用。成長計算はしない
  "stats": { "hp": 69, "str": 27, "def": 18, "int": 11, "mov": 4, "pp": 10 },
  "techs": ["猛撃", "烈火", "炎連拳"],     // 初期技を置き換え（techs.json の実在キーのみ）
  "skills": ["衝動", "闘争本能", "健脚のブーツ"], // parts.json の skill 名
  "moveType": "infantry"
}
```

### ステの考え方（重要）

`stats` は **スキル適用前の「素ステ」** を書く。スキルの常時statMod
（猛者の腕輪 STR+2 / 健脚のブーツ MOV+1 / 将軍の大盾 DEF+6,MOV-1 等）は
`applyPart` が上乗せするので、最終的な実効ステ＝ `stats` ＋ スキル補正 になる。
（本番と同じ「素ステ＋スキル」の合算経路）

### 仕様メモ

- **スキルは必ず `applyPart` 経由**で付与（衝動/反逆などカウンター系の `onAcquire`
  初期化も本番と同じルートを通る）。
- **技は `makeTech` 直**（techs.json に `onAcquire` 系の技は存在しないため）。
  ロスターの `techs` は初期技を**置き換える**（指定しなければ初期技のまま）。
- **枠制限は撤廃**（`_bonusSlots` を大きく盛る）。任意パーツを好きなだけ積める。
  そのぶん StatusScreen の枠数表示は大きく出るが想定通り。
- 不明な技名/スキル名/charId は `console.warn` を出してスキップする。

## お祭り面の差し替え

`src/maps/festival.json` は最小プレースホルダ（15体・殲滅目標・8枠デプロイゾーン）。
本番の24体規模マップはマップエディタ（`tools/map-editor.html`）で作って
このファイルを上書きすればいい。デプロイゾーンが8セル以上あれば自動配置が成立する。

## 調査メモ（実装時に確認した点）

- **§3.1**: `createPlayerUnit` は `initTechs` しか読まず `skills: []` 固定。
  units.json にスキル欄を足しても読まれない → 投入口（applyPart 経由）が必要。推定どおり。
- **§3.2**: `createPlayerUnit` は最終ステ直書きも成長計算もしていない（Lv1・冒険者固定）。
  どちらにせよ新規実装。今回は①直書きを採用。`route_v10` の最終ステ参照表は実ファイルとしては存在しなかった。
- **既存データの不整合（範囲外）**: units.json の初期技のうち `連撃 / 祈り / 盾殴り / 強弓 /
  よろめきの風` は techs.json にキーが無く、`createPlayerUnit` 時点で黙って落ちている。
  本番ユニットにも影響しているはず。お祭りロスターでは実在技のみ使用した。
