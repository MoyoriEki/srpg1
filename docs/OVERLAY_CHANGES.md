# タイルオーバーレイ改修

## 変更ファイル（3つ）

| ファイル | 変更内容 |
|----------|---------|
| `tools/map-editor.html` | 複数チップセット対応、Canvas描画化、ズーム16〜64px、マップ上限80x80 |
| `src/engine/map.js` | `getTilesets()` / `getTileOverlay()` API追加（旧形式自動互換） |
| `src/ui/MapView.jsx` | Canvas描画レイヤー追加、ハイライト分離（地形色→タイル画像→範囲ハイライト→ユニット） |

## 新ディレクトリ

```
public/tilesets/    ← ゲーム用チップセット画像をここに手動配置
```

## データ形式

### 旧形式（自動互換あり）
```json
{
  "tileset": {"src": "b02_緑地.png", "tileW": 48, "tileH": 48, "cols": 8},
  "tileOverlay": [[[1,2,3,...], ...], ...]
}
```

### 新形式
```json
{
  "tilesets": [
    {"src": "b02_緑地.png", "tileW": 48, "tileH": 48, "cols": 8},
    {"src": "interior_01.png", "tileW": 48, "tileH": 48, "cols": 10}
  ],
  "tileOverlay": [[[0,10001,2,...], ...], ...]
}
```

セル値のエンコード: `チップセットインデックス × 10000 + タイル番号`
- 旧形式の値（1〜N）はチップセット0として互換動作

## 使い方

### マップエディタ
- Tab2「画像オーバーレイ」でチップセットを複数追加可能
- 左パネルのチップセットリストで切り替え
- 右クリックでスポイト（チップセットも自動切替）
- JSONインポート時、旧tileset→tilesets[0]に自動変換

### ゲーム側
- `public/tilesets/` にチップセット画像ファイルを配置
  - 例: `public/tilesets/b02_緑地.png`
- マップJSONの `tilesets[].src` と一致するファイル名

## 描画階層（zIndex）

| 層 | zIndex | 内容 |
|----|--------|------|
| 地形色 | 0 | べた塗り（タイル画像がない場合のフォールバック） |
| タイルオーバーレイ | 1 | Canvas — 4レイヤー合成 |
| ハイライト | 2 | 移動/攻撃/回復範囲の半透明オーバーレイ |
| アイテムボックス | 3 | ◈マーカー |
| ユニット | 10 | チップ + HPバー |
| ダメージポップ | 20 | |
| ツールチップ | 45-50 | |
| バナー | 200 | PHASE表示 |

## 画質問題の修正

- エディタ: Canvas描画化でピクセルパーフェクト（旧: 32px divにbackground-sizeで縮小描画→ぼやけ）
- ゲーム: Canvas `imageSmoothingEnabled=false` + 48px等倍描画
